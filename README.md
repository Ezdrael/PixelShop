# Структура файлів

/your_project/
├── public/                <-- Публічна папка (доступна з браузера)
│   ├── index.php          <-- Фронт-контролер (всі запити йдуть сюди)
│   ├── css/
│   │   └── style.css
│   ├── js/
│   ├── img/
│   └── .htaccess          <-- Для перенаправлення всіх запитів на index.php
├── app/                   <-- Логіка програми
│   ├── controllers/
│   │   ├── HomeController.php
│   │   └── ProductController.php
│   ├── models/
│   │   ├── Product.php
│   │   └── User.php
│   └── views/
│       ├── layouts/
│       │   └── default.php  <-- Основний шаблон (містить header, footer)
│       ├── partials/
│       │   ├── header.php   <-- Ваша навігаційна панель
│       │   └── footer.php   <-- Ваш футер
│       ├── home/
│       │   └── index.php    <-- Головна сторінка
│       └── products/
│           ├── list.php     <-- Сторінка списку товарів
│           └── detail.php   <-- Сторінка детального опису товару
├── vendor/                <-- Бібліотеки Composer (якщо використовуєте)
├── .env                   <-- Змінні середовища
├── composer.json
└── README.md



Як це працює:
public/index.php (Фронт-контролер): Усі запити від браузера надходять сюди. Він аналізує URL, визначає, який контролер і метод потрібно викликати.

Контролер (HomeController.php, ProductController.php): Контролер виконує необхідну логіку (наприклад, отримує дані з моделі) і готує дані для представлення. Потім він "включає" (за допомогою require_once) відповідне представлення.

Модель (Product.php): Містить логіку для взаємодії з даними (у цьому прикладі - простий масив, але в реальності це була б база даних).

Представлення (app/views/home/index.php, app/views/products/list.php, app/views/products/detail.php): Це чистий HTML (з мінімальними PHP-вставками для виводу даних). Кожне представлення використовує ob_start() та ob_get_clean() для буферизації свого виводу, а потім передає його змінній $content.

Основний шаблон (app/views/layouts/default.php): Цей шаблон є обгорткою для всього сайту. Він включає header.php, footer.php та виводить $content, який є вмістом конкретного представлення.

Таким чином, ви розділяєте логіку, представлення та контролер, що робить ваш код більш організованим та легким для масштабування.
